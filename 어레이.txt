1. 자바의 데이터 타입 분류
    기본 타입 - 실제 값을 저장
    참조 타입 - 주소를 저장
    
2. 기본타입과 참조타입이 저장되는 위치가 다름.
    
3. 자바 메모리 구조(1)
 -----------------------------나무위키 : 검색 (jvm)

     0.  소스파일 -> 컴파일 -> 클래스파일(바이트 코드)
          (.java) 	 (javac)	   (.class)

     1. [ class Loader ] - 클래스파일(바이트코드)들을
        runtime data area 에 적재

     2. [ runtime data area ] -JVM 이 운영체제로 부터 
        할당받은 메모리 영역

     3. [ Garbage Collector ], GC - heap메모리 영역에서 사용하지 않은 객체들을 탐색 후 제거 

     4. [ execution engine ] - 명령어 단위로 실행하는 역할 

4. 자바메모리 구조(2)
     1) method area - 바이트코드 분석 후 저장, ( static ) 공유 영역
     2) heap area - 문자열, 배열, 객체같은 동적 생성된 데이터 저장되는 공간
     3) stack area - 잠깐 사용되는 데이터 저장
